# <a name="c-runtime-error-r6002"></a>C 런타임 오류 R6002
부동 소수점 지원이 로드 되지 않았습니다  
  
 필요한 부동 소수점 라이브러리가 연결 되지 않았습니다.  
  
> [!NOTE]
>  응용 프로그램을 실행 하는 동안이 오류 메시지를 발생 하는 경우 응용 프로그램 종료가 내부 문제가 있기 때문입니다. 이 오류에 대 한 여러 가지가 있지만 응용 프로그램의 코드의 결함 또는 특정 컴퓨터 프로세서에 대해 만들어지지 않은 하는 응용 프로그램 실행을 시도 하 여 발생 되는 경우가 많습니다.  
>   
>  이 오류를 해결하려면 다음 단계를 시도할 수 있습니다.  
>   
>  -   사용 하 여는 **앱 및 기능** 또는 **프로그램 및 기능** 페이지에 **제어판** 를 복구 하거나 프로그램을 다시 설치 합니다.  
> -   확인 **Windows Update** 에 **제어판** 소프트웨어 업데이트에 대 한 합니다.  
> -   응용 프로그램의 업데이트 된 버전을 확인 합니다. 문제가 지속 되는 경우 앱 공급 업체에 문의 합니다.  
  
 **프로그래머에 대 한 정보**  
  
 부동 소수점 라이브러리가 연결 되지 않았습니다 응용 프로그램에서이 오류가 발생할 수 있습니다. 이러한 원인 중 하나를 확인 합니다.  
  
-   에 대 한 형식 문자열을 `printf_s` 또는 `scanf_s` 함수에 부동 소수점 형식 사양이 들어 및 임의의 부동 소수점 값 또는 변수는 프로그램 들어 있지 않습니다. 이 문제를 해결 하려면 부동 소수점 형식 사양에 해당 하는 부동 소수점 인수를 사용 하거나 프로그램에서 부동 소수점 할당을 다른 위치에서 수행 합니다. 이렇게 하면 로드 되도록 부동 소수점 지원 됩니다.  
  
-   컴파일러는 부동 소수점 지원이 필요한 경우에 로드 하 여 프로그램의 크기를 최소화 합니다. 컴파일러 또는 검색할 수 없습니다 부동 소수점 연산 부동 소수점 형식 사양을 형식 문자열에 있으므로 필요한 부동 소수점 루틴을 로드 하지 않습니다. 이 문제를 해결 하려면 부동 소수점 형식 사양을 사용 하 고 부동 소수점 인수를 제공 하거나 프로그램에서 부동 소수점 할당을 다른 위치에서 수행 합니다. 이렇게 하면 로드 되도록 부동 소수점 지원 됩니다.  
  
-   혼합 언어 프로그램 프로그램이 링크 때에 C 라이브러리 FORTRAN 라이브러리 하기 전에 지정 했습니다. 다시 연결 하 고 마지막 C 라이브러리를 지정 합니다.
