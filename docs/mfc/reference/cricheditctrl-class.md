## <a name="remarks"></a>설명  
 "Rich edit 컨트롤"에 사용자 입력 수 및 텍스트를 편집 하는 창입니다. 텍스트 문자와 단락 형식을 할당할 수 있으며 포함 된 OLE 개체를 포함할 수 있습니다. Rich edit 컨트롤 텍스트 서식 지정에 대 한 프로그래밍 인터페이스를 제공 합니다. 그러나 응용 프로그램 사용자에 게 서식 지정 작업을 제공 하는 데 필요한 사용자 인터페이스 구성 요소를 구현 해야 합니다.

```  
void SetOptions(
    WORD wOp,  
    DWORD dwFlags);
```  
  
### <a name="parameters"></a>매개 변수  
 *wOp*  
 작업의 유형을 나타냅니다. 다음 값 중 하나입니다.  
  
- 지정 된 옵션을 설정 하는 ECOOP_SET *dwFlags*합니다.  
  
- 이 메서드를 사용 하 여 현재 옵션 ECOOP_OR 결합 *dwFlags*합니다.  
  
- ECOOP_AND 유지도로 지정 되는 현재 옵션만 *dwFlags*합니다.  
  
- 배타적 OR 연산을 논리적으로 ECOOP_XOR 현재 옵션에서 지정한 *dwFlags*합니다.  
  
 *dwFlags*  
 서식 있는 편집 옵션입니다. 플래그 값은 설명 섹션에 나열 됩니다.  
  
### <a name="remarks"></a>설명  
 옵션에는 다음 값의 조합 수 있습니다.  
  
- ECO_AUTOWORDSELECTION 자동 단어 선택 영역에는 다음 작업을 두 번 클릭 합니다.  
  
- 줄의 끝 문자를 입력할 때 10 자 텍스트 오른쪽에 자동으로 ECO_AUTOVSCROLL 스크롤합니다. 사용자가 ENTER 키를 누를 때 컨트롤이 0 위치로 다시 모든 텍스트를 스크롤합니다.  
  
- 마지막 줄에서 ENTER 키를 누를 때에 자동으로 ECO_AUTOHSCROLL 텍스트를 한 페이지 위로 스크롤합니다.  
  
- ECO_NOHIDESEL 편집 컨트롤의 기본 동작을 부정합니다. 기본 동작을 컨트롤 입력된 포커스를 잃고 컨트롤이 입력된 포커스를 받으면 선택 영역을 표시 하는 경우 선택 영역을 숨깁니다. ECO_NOHIDESEL를 지정 하는 경우 컨트롤에 포커스가 없는 경우에 선택한 텍스트 반전 됩니다.  
  
- ECO_READONLY 사용자를 입력 또는 편집 컨트롤의 텍스트를 편집할 수 없습니다.  
  
- 여러 줄 서식 있는 텍스트를 입력 하는 동안 사용자가 ENTER 키를 누를 때 캐리지 리턴을 삽입 수는 대화 상자에서 컨트롤을 편집 ECO_WANTRETURN 지정 합니다. 이 스타일을 지정 하지 않으면 ENTER 키를 부모 창의 기본 단추 (예를 들어 대화 상자에서 확인 단추)를 클릭 하면 유사한 rich edit 컨트롤의 부모 창에 명령을 보냅니다. 이 스타일에 편집 컨트롤을 한 줄에 적용 되지 않습니다.  
  
- ECO_SAVESEL은 컨트롤이 포커스를 잃었을 때 선택 항목을 유지 합니다. 기본적으로 포커스를 다시 얻은 경우 컨트롤의 전체 내용은 선택 됩니다.
